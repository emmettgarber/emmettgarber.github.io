Arrays and Hashes

Similarities and Differences

February 7, 2016

An array is a list of objects that are stored. The objects can be integers or strings, and are stored in an order. This order is important, as it is this order that one uses to access objects in array. The order starts at 0 and goes up by one for each object in an array. Lets look at how to make a an array of 3 object, and how to call each of them. To create the array: new_Array = [ "soccer", "pizza", 23]. To access soccer one would type: new_Array[0], to acces pizza: new_array[1], and to access 23 new_Array[2].

Hashes are similar in that they also store objects, which can be integers or strings, but unlike arrays this data is stored in pairs, not a list. For each pair, one can access the information by calling on the first item in the pair. To create a hash one would type: new_hash{ "first" => "me", "second" => "you", "third" => "them"}. To call on the 'me', one would type new_hash["first"] etc.

When would we use array vs a hash? You want to use arrays when you just have a list of things, lets say dog breeds dog_array["corgi", "retriever", "St Bernard"]. But lets say instead we want to know, the size of dog when we look up the breed, then we would want to use a hash: dog_hash{ "corgi" => "small", "retriever" => "large", "St Bernard" => "huge"}.
